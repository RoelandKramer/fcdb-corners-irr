# Inter-Rater Reliability — Corner Role Labelling

This folder contains everything needed to run the IRR study for the
defender- and attacker-role classifiers from the thesis.

Six raters (Roeland + 5 friends) each label a subset of 10 pre-selected
corner kicks through a Streamlit app. Their labels go to per-rater CSVs
which are then fed into `irr_analysis.py` for pairwise Cohen's kappa,
Fleiss' kappa across all raters, per-class agreement, and
marking-assignment agreement.

## Setup workflow (one-off)

### 1. Place the full-match videos

Put the 20 full-match source videos (4 camera angles × 5 matches) into
`source_videos/`, named by SciSports `file_match_id` and the camera type:

```
source_videos/
├── 2561505_espn.mp4         # FC Dordrecht vs Willem II
├── 2561505_mp4.mp4
├── 2561505_goal_left.mp4
├── 2561505_goal_right.mp4
├── 2561495_espn.mp4         # Almere City FC vs FC Dordrecht
├── 2561495_mp4.mp4
├── 2561495_goal_left.mp4
├── 2561495_goal_right.mp4
├── 7647_espn.mp4            # De Graafschap vs FC Dordrecht
├── ...
```

The `file_match_id` for each corner is in `corner_manifest.json` under
`match_id`. The pipeline already wrote the right IDs there.

### 2. Cut the 40 clips

Make sure ffmpeg is on your system PATH, then:

```
python cut_clips.py
```

This reads each corner's `clip_start_ms` / `clip_end_ms` from the
manifest and writes a 10-second clip for each (corner, camera) pair to
`videos/corner_XX_<camera>.mp4` (40 files = 10 corners × 4 cameras).

### 3. Test the app locally

```
pip install streamlit pandas scikit-learn
streamlit run app.py
```

Log in with one of the usernames (e.g. `RoelandKramer`) and the shared
password (defined at the top of `app.py`).

## Deploying to Streamlit Cloud

1. Push this folder (`interreliability_testing/`) to a GitHub repo —
   public is fine since you said privacy isn't a concern.
   Include the `videos/` folder so the app can find the clips.
2. On [share.streamlit.io](https://share.streamlit.io), create a new
   app pointing at this repo and `app.py`.
3. Add a `requirements.txt` next to `app.py` with:
   ```
   streamlit
   pandas
   scikit-learn
   ```
4. Deploy. Streamlit gives you a shareable URL.

> **Caveat about persistence on Streamlit Cloud.** The CSV that the app
> writes to `labels/` lives in the deployed app's ephemeral filesystem
> — when the app sleeps or redeploys, those files are wiped. The app's
> built-in *Download my labels (CSV)* button is therefore the
> authoritative way for friends to send their labels to you. Ask them
> to download the CSV at the end of their session and email it to you.
> You then place each `labels_<Friend>.csv` into your local
> `labels/` folder before running `irr_analysis.py`.

## Running the IRR analysis

After all CSVs are collected in `labels/`:

```
python irr_analysis.py
```

The script prints, for each side (defender and attacker):
- Pairwise Cohen's kappa for every rater pair
- Fleiss' kappa across all raters
- Per-class kappa (one-vs-rest, averaged across pairs)
- Marking-assignment agreement for defenders that everyone labelled MAN

It also writes `labels/irr_report.csv` with the full pairwise table.

Interpretation benchmarks:
- κ > 0.80 → almost perfect
- 0.60–0.80 → substantial
- 0.40–0.60 → moderate
- < 0.40 → fair or worse

`RoelandKramer`'s labels can be folded back into the main
`labeling_sheet_v2.csv` to expand the training set. The format of
`labels_RoelandKramer.csv` is compatible with the schema the FCDB
Analyser uses for coach corrections (one row per `(corner, player_team,
jersey)` with `role` and `marks`). A small merge script can match by
`corner_id` and overwrite the corresponding rows.

## Quick reference: roles being labelled

**Defenders** (pick one):
- `MAN` — tracks one specific attacker, follows their movement
- `ZONAL` — guards a fixed area of the box
- `SHORT` — stationed near the corner flag to deny the short option
- `COUNTER` — positioned high (around halfway line) for counter-attack cover

If `MAN`, also pick which attacker (jersey number) is being marked.

**Attackers** (pick one — describes the running line / movement role):
- `TARGET` — designed receiver of the cross
- `DECOY` — pulls defenders away from the receiver
- `STATIC` — stays in roughly the same position
- `SECOND_BALL` — positioned at the 16m line for rebounds
- `BLOCK_GK` — screens the goalkeeper
- `BLOCK_DEF` — blocks a specific defender from contesting

`DON'T KNOW` is also available on both sides — preferable to guessing.
